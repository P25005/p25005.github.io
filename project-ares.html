ARES: AI-Powered Race Engineer & Telemetry Analyst

Una aplicación de escritorio full-stack que fusiona el análisis complejo de datos de telemetría con IA generativa (Google Gemini) para ofrecer ingeniería de pista en tiempo real para sim racing.

Descripción General

ARES (AI Race Engineer System) es un prototipo de software comercial diseñado para democratizar la ingeniería de competición en Assetto Corsa Competizione.

El problema en el sim racing de alto nivel es que el análisis de telemetría (MoTeC) y el ajuste del "setup" del coche requieren conocimientos de ingeniería mecánica. ARES soluciona esto actuando como un ingeniero de pista virtual. La aplicación ingiere datos crudos de telemetría (.ld), lee la configuración actual del vehículo (.json) y procesa el feedback en lenguaje natural del piloto.

Utilizando una arquitectura híbrida y la API de Google Gemini, ARES "traduce" gigabytes de datos complejos en instrucciones de ajuste precisas y comprensibles, manteniendo una memoria conversacional para iterar sobre el rendimiento del vehículo sesión tras sesión.

Nota sobre el Estado del Proyecto: Este es un proyecto "escaparate" de un producto en fase de pre-comercialización. El código fuente es privado, pero esta entrada detalla la arquitectura y las soluciones técnicas implementadas.

Highlights Técnicos

  • Arquitectura de Escritorio Híbrida Desarrollo de una aplicación nativa sin navegador visible, encapsulando un frontend moderno en Angular (TypeScript) y un backend robusto en Python (FastAPI) mediante PyWebView. Esto permite el acceso al sistema de archivos local necesario para leer telemetría mientras se ofrece una UX reactiva.
  • Pipeline de Datos Complejo Implementación de ldparser en Python para la extracción y procesamiento matemático de ficheros de telemetría binarios de MoTeC. El sistema analiza histogramas de suspensión, temperaturas de neumáticos y actividad del ABS para generar insights estructurados antes de enviarlos a la IA.
  • Ingeniería de Prompts Avanzada & Memoria Diseño de un sistema de interacción con el LLM (Google Gemini) que va más allá de una simple consulta. ARES construye un contexto rico (RAG-lite) combinando datos duros (telemetría/setup) con datos blandos (feedback del piloto). Crucialmente, implementa memoria de sesión iterativa, permitiendo a la IA recordar cambios anteriores y afinar sus sugerencias como un ingeniero humano real.
  • Full-Stack Moderno Comunicación eficiente entre la UI de Angular y el microservicio local de FastAPI para manejar cargas de trabajo asíncronas pesadas (análisis de IA) sin congelar la interfaz de usuario.

Stack Tecnológico

Frontend: Angular, TypeScript, SCSS.
Backend: Python 3, FastAPI, Uvicorn.
AI & ML: Google Gemini API, Prompt Engineering.
Data: ldparser (Binary Parsing).
Desktop: PyWebView (Native wrapper).

Resumen

  • Ingeniería de pistas virtual con IA.
  • Análisis de binarios MoTeC (.ld).
  • APP Híbrida (Web tech en Desktop).
  • Memoria de sesión contextual.
